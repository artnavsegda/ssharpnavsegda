In order to demonstrate the OAuth process, this OAuth Client needs to be hooked up with a Resource Owner, an Authorization Server, and a Protected Resource. Auth0 will provide all of these components.

Go here to make an Auth0 account. Signing up will take you to a dashboard with an "Applications" button on the sidebar. Click there and you will see that there is already a default application registered; click it and then go to the "Settings" tab for this app.

You may want to rename the application "CWS OAuth Client Example" or similar, but the important thing is to scroll down and paste the following URL into the "Allowed Callback URLs" field:

{@callbackUrl}

In addition,

  • Set the Application Type to Regular Web Application
  • Set the Token Endpoint Authentication Method to Post

Scroll back up and you'll see three fields that Auth0 has generated for the app: A Domain, Client ID, and a Client Secret. These three values need to be communicated to the client somehow so that the client knows where to find the Authorization Server and how to authenticate itself to it.
